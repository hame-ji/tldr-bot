Title: Automated Code Review Workflows via GitHub Copilot CLI

TL;DR: Integrating AI coding agents into Git hooks automates engineering best practices and enforces strict code quality checks prior to final human review.

Key points:
- GitHub Copilot's CLI mode can be used to construct an agentic workflow that automatically audits code during the commit process.
- AI review agents are constrained using strict rules, predefined project memory, and bounded terminal commands to ensure high-quality code generation.
- Enabling verbose mode (using the `-x` flag) in Git hook shell scripts provides essential visibility for debugging the automated actions taken by the AI agent.
- Developers can fine-tune the agent's strictness, such as instructing it to fix reported violations automatically without blocking the commit for lower-severity warnings.

Why it matters:
- Delegating routine code audits to CLI-based AI agents accelerates development by programmatically enforcing baseline best practices, reserving human effort for final verification.

Evidence:
- The system is implemented using tools like Copilot CLI and `jq` to evaluate diffs, successfully catching configuration violations like unauthorized `.local` file commits.

Caveat:
