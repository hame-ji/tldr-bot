Title: How AI Coding Agents Make Free Software Relevant Again

TL;DR: AI coding agents enable users to practically exercise the freedoms of open-source software by acting as proxies that can read, understand, and modify code on their behalf.

Key points:
- The shift to SaaS and the corporate rebrand to "open source" previously sidelined the original "free software" movement by locking source code on vendor servers where users could not modify it.
- Customizing closed SaaS platforms today requires brittle workarounds, such as reverse-engineered APIs and plaintext authentication, because agents cannot directly alter locked source code.
- AI agents bridge the technical gap, allowing even non-technical users to endlessly customize software directly, provided the underlying codebase is open and accessible.
- As users rely more heavily on AI agents, rigid proprietary software will increasingly be treated as an obstacle, driving a shift toward open systems that agents can easily integrate with and modify.

Why it matters:
- The primary buying criterion for software is expected to shift from operational convenience to agent-customizability, severely threatening SaaS companies that rely solely on switching costs rather than deep structural moats.

Evidence:
- Automating a Sunsama workflow using an agent required six layers of workarounds, including manual iOS shortcut creation and reliance on a reverse-engineered API, whereas modifying a free software alternative would have taken the agent minutes.
- The creator of Tailwind CSS reported a 40% drop in documentation traffic, an 80% drop in revenue, and a 75% reduction in engineering staff as AI coding usage increased.

Caveat:
- While agents empower users, they threaten the sustainability of the open-source ecosystem by severing the user-maintainer feedback loop and flooding projects with low-quality AI-generated contributions.
