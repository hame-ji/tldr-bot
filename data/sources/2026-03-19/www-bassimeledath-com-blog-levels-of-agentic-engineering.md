**Title:** The8 Levels of Agentic Engineering  

**Key Points**  
1. **Levels 1‑2 – Tab‑complete & Agent IDE:** Early AI assistance (e.g., Copilot, Cursor) provides inline code completion and chat‑linked multi‑file edits, but usefulness is limited by the model’s context window and the need for manual planning.  
2. **Level 3 – Context Engineering:** Focuses on maximizing information density in prompts (system prompts, rules files, tool descriptions, conversation history) so the model receives the right context and avoids noise; the practice has evolved from filtering bad context to ensuring relevant context is present when needed.  
3. **Level 4 – Compounding Engineering:** Introduces a plan‑delegate‑assess‑codify loop; lessons learned from each session are codified (e.g., updating CLAUDE.md or maintaining a docs/ folder) so future sessions benefit from accumulated knowledge.  
4. **Level 5 – MCP and Skills:** Gives the LLM executable capability via Model‑Context‑Protocols (MCPs) or custom skills (database access, API calls, CI pipelines, browser testing, etc.), enabling the model to act on the codebase rather than only reason about it.  5. **Level 6 – Harness Engineering & Automated Feedback Loops:** Builds an environment where agents can verify their own work (type checks, tests, linters, pre‑commit hooks, security boundaries) and iterate autonomously; backpressure mechanisms let agents detect and correct mistakes without human intervention.  
6. **Level 7 – Background Agents:** With reliable context, tooling, and feedback loops, agents can plan and execute tasks asynchronously (e.g., Ralph loop, Dispatch orchestrator, cloud‑hosted sandboxes); different models can be specialized for implementation, review, or research, and agents can be triggered from CI/CD pipelines for continuous self‑improvement.  
7. **Level 8 – Autonomous Agent Teams:** Removes the central orchestrator; agents coordinate directly, claim tasks, share findings, and resolve dependencies in a peer‑to‑peer fashion. Early experiments show promise but also highlight coordination challenges, hierarchy needs, and current economic/performance limits.  

**Why It Matters**  
Understanding these levels clarifies where current AI‑assisted coding practices sit and what incremental investments (context management, skill/tool building, feedback loops, orchestration) yield the biggest productivity gains. It also highlights that individual output is tightly coupled to teammates’ levels—raising the team’s overall capability amplifies the benefits of advancing models, while neglecting foundational work (context, harnessing, skills) undermines higher‑level automation. This framework helps engineers prioritize effort, design effective AI‑augmented workflows, and anticipate the evolving role of humans in software development.
